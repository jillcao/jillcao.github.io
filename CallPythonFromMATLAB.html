Call Python from MATLAB

Tags: user research, user interviews, usability testing

Time frame: 2013 - 2014

Project goal: The goal of this product is to enable users to call Python libraries from within MATLAB.

Challenge: The overall design challenge was how to make an intuitive interface that best handled the confusion in users arisen from having to work with two languages. MATLAB and Python differ in their syntaxes, data types, and programming workflows. As a result, a user can become confused when attempting to call a Python module from MATLAB. For example, the dictionary datatype is unique to Python. A user may wonder what to do when he/she is trying to call a Python module that expects a dictionary as input because MATLAB does not support dictionaries.

Approach: I led interviews to understand user's mental model of how MATLAB data types mapped to Python data types. In addition, I led usability studies to understand when users expect MATLAB to automatically convert a MATLAB data type to a Python data type, what steps they expect in a workflow to call a Python module.

UX methodGoal
Interviews To understand user's mental model of how MATLAB data types mapped to Python data types.
Usability testing (Wizard of Oz) To understand
  • when users expect MATLAB to automatically convert a MATLAB data type to a Python data type.
  • what steps users expect in the workflow of calling a Python module from MATLAB and getting the results back in MATLAB.
  • what obstacles users run into in that workflow.
 

How I worked with my team: I joined the project when the team had already started designing without formal user research. Instead of proposing to start over with user research which would have appeared as a back step to the team, I leveraged their request to "help us do some usability testing" to really understand the team’s needs. After interviewing the team, I discovered that the team wanted to better understand user's mental model of data type conversion. I found this to be a better fit for user interviews than usability testing so I helped the team conduct the interviews with internal and external customers. Using data gathered from the interviews, we refined the design and tested it through Wizard of Oz usability testing without having to implement the design.

Results and impact: I helped shape the design and gain buy-in from upper management and other stakeholders.

  • Helped prioritize development effort around the frequency of data type usage based on user interview finding.
  • Streamlined user workflows based on user’s mental model.
  • Helped team gained design buy-in from CEO and VP-level management at design reviews by providing solid usability data to back up design proposals.
  • Upper management praised the approach of grounding a design proposal with real user data at design reviews and requested more of it.
